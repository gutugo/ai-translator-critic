Роль: Ты QA Automation Lead, специализирующийся на UI-тестировании с помощью Cypress.
Контекст: 
- У нас есть Flask-приложение "AI Translator & Critic", доступное по адресу http://localhost:5000 (или через перенаправленный порт Codespaces).
- Это приложение делает два POST-запроса к внешнему API: один для перевода (модель Qwen/Qwen3-VL-30B-A3B-Instruct) и один для оценки (модель claude-sonnet-4-5-20250929).
Задача:
Создай набор UI-тестов для проверки ключевого пользовательского сценария в приложении.
Технологии и структура:
1. Фреймворк: Cypress.
2. Файл: cypress/e2e/translator_critic.cy.js.
Обязательные требования к тестам:
1. MOCKING СЕТЕВЫХ ЗАПРОСОВ (Критически важно):
- Используй **cy.intercept()** для перехвата (mocking) всех запросов к внешнему API: https://api.mentorpiece.org/v1/process-ai-request
- **Mock 1 (Перевод):** Перехвати запрос, где тело содержит model_name: "Qwen/Qwen3-VL-30B-A3B-Instruct", и заставь его немедленно ответить с кодом 200 и телом: {"response": "Mocked Translation: The sun is shining."}
- **Mock 2 (Оценка):** Перехвати запрос, где тело содержит model_name: "claude-sonnet-4-5-20250929", и заставь его немедленно ответить с кодом 200 и телом: {"response": "Mocked Grade: 9/10. Fluent and accurate."}
2. Основной Сценарий ("Успешный Перевод и Оценка"):
- Тест должен начинаться с cy.visit('/')
- Ввод текста: Введи "Солнце светит." в <textarea>.
- Выбор языка: Выбери "Английский" из <select>.
- Нажатие кнопки "Перевести".
- Проверка 1: Убедись, что **Mock 1** (запрос на перевод) был отправлен.
- Ожидание перевода: Проверь, что в блоке результатов появился текст: "Mocked Translation: The sun is shining."
- Нажатие кнопки "Оценить при помощи LLM-as-a-Judge"/
- Проверка 2: Убедись, что **Mock 2** (запрос на оценку) был отправлен.
- Ожидание оценки: Проверь, что в блоке оценки появился текст: "Mocked Grade: 9/10. Fluent and accurate."

3. Сценарий обработки ошибок (API Failure Mock):
- Создай отдельный тест, который мокает API (для обеих моделей) для ответа с кодом **500 Internal Server Error**.
- Проверь, что после нажатия кнопки приложение отображает пользователю **сообщение об ошибке**, а не падает.
Действия:
В директории tests/ui:
1. Сгенерируй полный код для файла **cypress/e2e/translator_critic.cy.js**.
2. Включи подробные комментарии в коде, объясняющие, как работает **cy.intercept()** и как проверяется **асинхронный контент**.
3. Предоставь инструкции по установке Cypress (NPM команда) и запуску теста в Codespaces.
